# 用户培训与知识迁移技能

## 适用场景

- 产品上线或功能升级，需要对用户/客户/内部团队培训。
- 交付项目给运维、客服、业务团队。
- 构建持续的知识库与反馈闭环。

## 前置准备

- 明确受训对象（运营、客服、终端用户、合作伙伴）。
- 收集培训目标、关键任务、工具、时间窗口。
- 准备培训材料模板（PPT、操作手册、视频脚本）。

## 操作步骤

1. **培训需求分析**
   - 调研受训者角色、技能水平、可用时间。
   - 定义学习目标、考核标准、实践环节。
2. **内容规划**
   - 制作培训大纲：概览、角色任务、操作步骤、常见问题、升级与支持。
   - 准备操作手册、FAQ、流程图、视频演示。
   - 为关键流程提供演练环境与测试账号。
3. **培训执行**
   - 安排培训形式：线上直播、现场 Workshop、自学课程。
   - 记录培训过程、答疑、改进建议。
   - 提供测试或考核确保掌握度。
4. **知识库建设**
   - 将文档、视频、FAQ、操作演示上传至知识库。
   - 建立版本管理与更新机制。
5. **反馈与支持**
   - 收集受训者反馈，调整培训内容。
   - 指定后续支持渠道（工单、IM 群、热线）。
   - 监控上线后问题，及时更新知识库。

## 质量校验

- 培训资料完备，覆盖所有关键场景与疑难点。
- 受训者考核通过率、反馈满意度达到目标。
- 知识库可访问、版本清晰、更新及时。

## 失败与回滚

- 培训效果不佳：增加辅导、调整内容、补充材料。
- 知识库失效或过期：恢复旧版本，重新审核后发布。
- 上线后问题集中：启动专项支持组，快迭代培训内容。

## 交付物

- 培训计划、日程、参会记录。
- 用户手册、操作指南、FAQ、演示视频。
- 知识库链接、后续支持流程、反馈报告。
