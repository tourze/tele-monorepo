# 观测复盘技能

## 适用场景

- 上线后定期检查观测体系效果。
- SLO/SLA 执行情况需要评估与调整。
- 复盘报警噪音、漏报、响应效率。

## 前置准备

- 收集 RED/USE 指标、业务 KPI、报警记录、值班日志。
- 获取之前制定的观测计划与 Runbook。
- 明确评审周期（周报、月度、迭代末）。
- 汇总最近一次 `method-business-impact-engineering` 输出的 NSM、输入指标、护栏、单位经济与实验假设。

## 操作步骤

1. **指标复盘**
   - 分析关键指标趋势：成功率、延迟、错误率、资源占用。
   - 对比阈值、基线、SLO/SLA、护栏与单位经济目标达成情况。
2. **报警评估**
   - 统计报警数量、平均响应时间、误报率、漏报事件。
   - 分类报警（P0/P1/P2），识别噪音源。
3. **事件回顾**
   - 选择典型事件，检查检测、响应、恢复、交接情况。
   - 更新 Runbook 中的步骤是否有效。
4. **优化动作**
   - 调整阈值、指标、仪表盘布局，更新护栏、单位经济成本监控。
   - 补充缺失指标或 tracing 点。
   - 更新报警路由、升级策略、值班安排。
5. **结论与计划**
   - 输出复盘报告、行动项、负责人、截止时间。
   - 在下一周期跟踪执行情况。

## 质量校验

- 报告包含指标趋势、报警分析、事件回顾、改进计划。
- SLO/SLA 调整有依据，获相关负责人批准。
- 行动项有 Owner、时间、验证方式。

## 失败与回滚

- 指标缺失：补采集后再复盘，记录原因。
- 行动项未执行：升级到负责人，重新计划。
- 调整引发误报/漏报：恢复旧阈值，重新评估。

## 交付物

- 观测复盘报告、改进清单。
- 更新后的仪表盘链接、报警策略。
- 行动项跟踪记录。
