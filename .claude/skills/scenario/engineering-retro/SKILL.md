# 工程复盘技能

## 适用场景
- 对会话或模块进行复盘，评估质量门执行、协作效率、知识沉淀。
- 形成行动计划、观察项、推广建议。

## 操作步骤
1. **范围确认**
   - 明确复盘对象：会话、模块、任务。
   - 收集操作记录、日志、提交、验证结果。
2. **核心洞察**
   - 总结成果、暴露问题，回答 Linus 三问。
   - 进行根因分析（技术债、流程缺失、工具限制）。
3. **模式提炼**
   - 区分通用模式与本地知识。
   - 描述适用范围、价值、限制。
4. **协作评估**
   - 记录亮点与改进点（人机协作、交接、信息流）。
5. **行动计划**
   - 即时行动（修复、补文档、回滚）。
   - 长期建议：命令基线更新、待观察、推广、待决策。
   - 使用图表/表格呈现计划与优先级。

## 质量校验
- 报告结构完整，洞察清晰，证据充分。
- 行动项具备 Owner、优先级、时间、成功标准。
- 建议与团队规范对齐，避免孤例泛化。

## 失败与回滚
- 复盘空泛：补充数据、具体案例。
- 行动项无人负责：指定 Owner 或升级处理。
- 建议冲突：记录争议，提请评审。

## 交付物
- 工程复盘报告（摘要、洞察、行动项）。
- 长期建议与观察计划。
- 更新或待更新的命令/技能列表。
