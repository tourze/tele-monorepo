# 安全评审技能

## 适用场景
- 新功能上线、架构变更、第三方集成前需要安全评估。
- 处理隐私数据、支付、认证等高风险模块。
- 准备安全审批材料、整改计划与复审。

## 前置准备
- 收集功能说明、数据流、接口文档、依赖列表。
- 获取数据分类、合规要求（GDPR、等保、隐私政策）。
- 明确安全评审委员会（安全、法务、合规、业务负责人）。

## 操作步骤
1. **威胁建模**
   - 使用 STRIDE/LINDDUN 分析威胁与隐私风险。
   - 绘制数据流图（DFD），标注信任边界、资产、处理节点。
2. **风险分析**
   - 列出潜在威胁、影响、发生概率、现有控制措施。
   - 计算风险等级（高/中/低）或 RPN。
3. **控制措施评估**
   - 检查鉴权授权、输入输出校验、日志、监控、加密策略。
   - 验证第三方接口安全条款、合同、密钥管理。
4. **审批材料**
   - 整理安全矩阵、风险评估、整改计划、测试结果。
   - 提交安全评审，记录结论与要求。
5. **整改与复审**
   - 制定整改任务，指定 Owner、截止时间。
   - 完成后重新测试（渗透、扫描、SAST/DAST）。
   - 复审通过后更新 Runbook 与文档。

## 质量校验
- 威胁建模完整，覆盖数据流与信任边界。
- 评审记录包含决策、整改项、状态。
- 敏感数据处理有加密/脱敏/访问控制措施。
- 安全测试通过（渗透、扫描、单元安全测试）。

## 失败与回滚
- 高风险未整改：禁止上线，制定补救计划。
- 评审信息缺失：补充数据流、接口、日志策略。
- 整改延误：升级至安全负责人，评估上线计划。

## 交付物
- 安全评审报告（威胁建模、风险矩阵、整改计划）。
- 评审会议纪要、审批结果。
- 更新后的安全控制文档、测试报告。
