# 变更管理技能

## 适用场景

- 需求、设计、发布计划出现调整。
- 管理 CAB（Change Advisory Board）流程，确保变更可控。
- 输出发布说明、沟通计划、回滚策略。

## 前置准备

- 建立变更申请模板：背景、原因、影响、风险、回滚方案。
- 明确审批人、CAB 成员、会议节奏。
- 与 PMO、运维、业务、QA 对齐变更窗口。

## 操作步骤

1. **变更申请**
   - 申请人填写表单，附上影响分析、成本、收益。
   - 编号并记录在变更日志，设置优先级。
2. **影响评估**
   - 分析对范围、时间、成本、质量、风险的影响。
   - 评估依赖（系统、团队）、需要的测试覆盖。
3. **审批流程**
   - 小变更：快速审批（PM + 负责人）。
   - 大变更：提交 CAB，会议讨论决策，记录结论与条件。
4. **执行与沟通**
   - 更新计划、报价、资源分配。
   - 输出发布说明：变更内容、影响范围、操作步骤、回滚。
   - 通知干系人（邮件、群、会议纪要）。
5. **收尾与复盘**
   - 变更实施后验证结果，记录 KPI。
   - 更新文档（PRD、技术方案、Runbook）。
   - 回顾变更流程，提高响应效率。

## 质量校验

- 每项变更有完整记录：申请、评估、决策、执行、结果。
- 发布说明明确，包含操作步骤、验证、回滚。
- 干系人通知到位，无信息缺口。
- 变更后指标符合预期，或记录偏差原因。

## 失败与回滚

- 未审批变更：禁止实施，追溯责任。
- 影响评估不充分：暂停，重新评估并补充测试计划。
- 变更失败：立即执行回滚策略，启动事故流程。

## 交付物

- 变更申请与评估表、CAB 决议记录。
- 发布说明、沟通记录。
- 变更后验证报告与回顾总结。
