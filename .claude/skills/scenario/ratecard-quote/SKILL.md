# Rate Card 报价技能（生命周期驱动）

## 适用场景

- 根据客户需求输出完整报价单，覆盖项目管理 → 需求 → 设计 → 开发 → 测试 → 部署 → 运维全生命周期。
- 在开发阶段按功能模块拆分任务，提供资源类型、人日、成本明细。
- 需要引用最新 Rate Card，并在变更/延期时进行重审。

## 前置准备

- 引用 Value Brief 的商业目标、RICE/WSJF 评分、单位经济假设，确保报价与优先级一致。
- 收集 Rate Card（角色、职责、单价、有效期），若缺失需立即向客户索取。
- 澄清项目范围、阶段目标、核心功能模块、假设与风险。
- 准备标准表格列：`项目阶段 | 功能模块/任务 | 描述 | 负责资源 | 单价 | 数量（人/日） | 金额`。

## 操作步骤

1. **需求理解与阶段划分**
   - 将项目切分为：项目管理、需求分析、产品设计、开发、测试、部署、运维支持等阶段。
   - 对每阶段列出关键任务与交付物，确保覆盖 6D 生命周期。
2. **开发阶段模块化**
   - 列举核心功能模块：功能名称、范围、技术方案简述。
   - 对每模块拆解任务（后端接口、前端页面、联调等），明确依赖关系。
3. **资源评估**
   - 为每个任务匹配 Rate Card 中的角色（如项目经理、后端工程师、测试工程师）。
   - 估算人日数，考虑缓冲（通常 10%-20%）与并行约束。
4. **成本计算**
   - 计算金额 = 单价 × 人日；阶段小计与项目总计需显式列出。
   - 如需税费或管理费，列明计算方式。
5. **假设与风险**
   - 说明报价假设（数据可用性、环境、第三方接口等）。
   - 标注触发重审的条件：范围变更、延期、Rate Card 更新。
6. **有效期与沟通记录**
   - 指定报价有效期（如 30 天）。
   - 记录干系人评审意见与确认结果。

## 输出示例（表头）

```md
| 项目阶段 | 功能模块/任务 | 描述 | 负责资源 | 单价（元/人天） | 数量（人/日） | 金额 |
| -------- | ------------- | ---- | -------- | --------------- | ------------- | ---- |
| 需求分析 | 需求澄清会 | 与业务梳理 MVP 范围 | 产品经理 | 2000 | 3 | 6000 |
```

## 质量校验

- 所有阶段均有条目，开发阶段细化到模块。
- 人日、单价、金额计算准确，阶段小计 + 总计匹配。
- 与 PRD、项目计划、技术方案对齐，无遗漏或重复计算。
- 列出风险、假设、有效期，提供重审触发条件。

## 失败与回滚

- 估算偏差：记录实际 vs 估算，更新模型或 Rate Card。
- 篇章结构混乱：按生命周期重排，确保层次清晰。
- 数据源不完整：暂停发布，待客户补充 Rate Card 或需求后继续。

## 交付物

- Markdown 报价单（必要时导出 Excel/PDF）。
- Rate Card 附录或链接、假设与风险说明。
- 审核记录、确认邮件或会议纪要。
