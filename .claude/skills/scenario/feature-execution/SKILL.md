# 特性实施技能

## 适用场景

- 依据 FRD/技术方案执行特性开发，确保任务闭环可追溯。
- 遵循状态机：解析 → 预检 → 执行 → 质量门 → 自愈 → 报告。
- 需要记录质量门结果、阻塞与回滚策略。

## 关联技能

- `scenario/feature-design`（理解 FRD 结构与约束）
- `scenario/quality-gates`（执行格式/静态分析/测试/构建）
- `scenario/feature-validation`（验证报告与修复任务对齐）
- `method/context-snapshot`（长流程记录快照）
- 语言/工具技能按任务涉及的技术栈加载

## 前置准备

- 获取最新的 Value Brief、指标链路、灰度放量与回滚方案（来自 `method-business-impact-engineering`），确认尚未过期。
- 最新 FRD（任务分解、验收标准、依赖、风险）。
- 相关规则（语言规范、质量门命令、架构红线）。
- 环境准备：依赖服务、测试数据、工具链。

## 操作步骤

1. **解析与预检**
   - 读取 FRD 及 Value Brief，建立任务清单并标注对应指标与假设。
   - 快速检查现有架构与 FRD 要求是否冲突。
   - 明确本次迭代需要交付的埋点、特性开关、回滚脚本位置。
2. **逐项执行**
   - 按任务分解逐项实施，更新任务状态。
   - 遇到架构冲突/信息缺失，暂停并上报。
3. **质量门**
   - 每个任务完成后运行格式、静态分析、测试、依赖、构建（如适用）。
   - 自动修复或重构后执行二次验证。
4. **自愈与升级**
   - 质量门失败时尝试自愈（≤3 次）。
   - 仍失败则标记任务为需返工，记录原因与建议。
5. **报告**
   - 汇总已完成任务、阻塞项、质量门结果、指标/单位经济影响。
   - 更新 FRD 的验证报告或生成独立执行报告，并回填 Value Brief 执行状态。

## 质量校验

- 执行结果与 FRD 任务状态一致。
- 质量门日志完整，无忽略或跳过。
- 阻塞项、返工项记录 Owner、原因、后续计划。
- 变更符合架构红线，无越权修改。

## 失败与回滚

- 质量门持续失败：暂停执行，升级给设计或架构负责人。
- 实施偏离 FRD：回退改动，重新对齐需求。
- 紧急阻塞：更新 FRD 状态，通知干系人，制定恢复计划。

## 交付物

- 执行报告（任务清单、质量门结果、阻塞项）。
- 更新后的 FRD 验证报告。
- 合规的提交/补丁与回滚说明。
