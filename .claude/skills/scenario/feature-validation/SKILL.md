# 特性验证技能

## 适用场景

- 发布前对照 FRD 验收标准确认特性达标。
- 生成修复任务、回归计划、报告。

## 前置准备

- Value Brief、指标链路、护栏指标、实验/灰度计划（来自 `method-business-impact-engineering`）。
- 最新 FRD（质量验收标准、任务分解、验证报告部分）。
- 质量门命令与目标路径。
- 测试数据、环境、监控指标。

## 操作步骤

1. **读取标准**
   - 提取 FRD 验收标准（功能、非功能、监控）与 Value Brief 中的指标、单位经济目标。
   - 确认任务分解与实际实现一致。
2. **执行质量门**
   - 运行格式/静态分析/测试/依赖/构建等命令。
   - 对比 FRD 与 Value Brief 中的指标阈值（性能、错误率、单位成本、护栏等）。
3. **差距分析**
   - 记录失败项、偏差原因、影响范围及指标/单位经济偏差。
   - 分类修复任务（R01/R02...），指定 Owner 与截止时间。
4. **报告生成**
   - 更新 FRD 验证报告章节或输出独立报告。
   - 提供状态总结：通过 / 需修复 / 阻塞。
5. **复核与交接**
   - 与相关团队确认结果。
   - 如需返工，回流至实施流程（参见 `scenario/feature-execution`）或重新进入设计阶段。

## 质量校验

- 所有质量门命令执行成功且结果记录。
- 验收标准逐条对照，有结论与证据。
- 修复任务编号、描述、责任人、优先级清晰。
- 报告与 FRD 状态同步。

## 失败与回滚

- 质量门失败：阻止上线，记录为阻塞，发布修复计划。
- 标准缺失：补充验收标准后再执行。
- 报告不一致：校对并更正差异，必要时重跑质量门。

## 交付物

- 验证报告（日志、结果、修复清单）。
- 更新后的 FRD 文档。
- 修复任务列表与跟踪链接。
