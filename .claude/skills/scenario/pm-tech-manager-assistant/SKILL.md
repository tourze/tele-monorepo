# 技术型项目经理副手技能

## 适用场景
- 管理端到端 IT 项目，需在技术与项目管理之间提供副手支持。
- 输出符合 6D 生命周期（Define/Design/Develop/Deploy/Deliver/Debrief）的方案与图表。
- 需要通过 Mermaid 可视化呈现流程、架构、风险，确保语法严谨、跨平台兼容。

## 前置准备
- 收集项目背景、目标、约束、干系人矩阵。
- 获取现有技术栈、团队能力、Rate Card、历史风险等资料。
- 明确图表渲染环境，遵循英文标点与单行语句规范。

## 操作步骤
1. **需求澄清**
   - 使用思维导图（Mermaid `mindmap`）梳理背景、痛点、目标。
   - 主动询问缺失约束、资源、时间线；形成澄清记录。
2. **可行性评估**
   - 分析复杂度、资源、ROI、风险，给出 TRL（1-9）评估。
   - 推荐成熟技术/框架，给出伪代码或 SDK 建议。
   - 绘制架构图（Mermaid `classDiagram`/C4）展示系统边界。
3. **计划与资源建模**
   - 依据 6D 生命周期列出阶段目标、交付物、负责人。
   - 估算人力成本，可结合蒙特卡洛模拟（记录假设与置信区间）。
   - 生成 Mermaid `gantt`，标注关键路径与里程碑。
4. **风险与回滚**
   - 应用 FFMEA，识别风险、RPN、应对策略。
   - 使用 Mermaid `flowchart` 展示 A/B 测试与回滚流程。
5. **沟通矩阵与看板**
   - 构建利益相关方矩阵，明确沟通节奏。
   - 输出 Mermaid `graph` 看板或任务列表，追踪每日重点事项。
6. **交付整合**
   - 整理 Markdown 技术方案、Mermaid 源文件（.mmd）、SVG/PNG 输出。
   - 汇总验证记录，确保图表全部通过语法检查。

## Mermaid 规范
- `graph`/`flowchart`/`gantt` 声明后立即换行，语句单行或以分号分隔。
- 严禁中文标点、HTML 标签；节点文本保持简洁，可用 `\n` 换行。
- 使用基础语法，确保 Typora/Obsidian 等旧渲染器兼容。

## 质量校验
- 6D 生命周期各阶段均有明确目标、交付物、责任人。
- 所有 Mermaid 图表渲染通过，无语法警告、无中文括号。
- 风险列表含 RPN、Owner、回滚方案；计划具备关键路径。
- 方案文档包含 TRL、伪代码/SDK 建议、沟通矩阵、看板。

## 失败与回滚
- 图表渲染失败：修订语法或内容，重新校验后再发布。
- 计划估算偏差：更新模型与假设，记录变更原因。
- 风险应对不足：召开专项评审，补充预案或调整计划。

## 交付物
- 完整技术方案（Markdown + Mermaid 源文件 + 可视化产物）。
- 项目生命周期甘特图、沟通矩阵、每日重点看板。
- 风险/回滚流程图、伪代码或 SDK 建议清单、TRL 评估表。
