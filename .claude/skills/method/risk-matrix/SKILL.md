# 风险矩阵技能

## 适用场景
- 功能设计、开发、发布阶段需要评估风险。
- 事故复盘或技术方案评审需要量化影响。
- 制定缓解措施与监控指标，确保风险可控。

## 前置准备
- 获取项目背景、范围、依赖、历史事故数据。
- 准备风险登记模板（Markdown/Excel），包含严重度、概率、Owner、到期。
- 熟悉 RED/USE 指标体系，用于观测。

## 操作步骤
1. **识别风险**
   - 从业务、技术、运营、安全、合规等维度搜集风险。
   - 结合历史事故、技术债、外部依赖、团队能力。
2. **量化评估**
   - 构建 5×5 矩阵：严重度（1-5）、概率（1-5）。
   - 优先级 = 严重度 × 概率；划分为 P1/P2/P3/P4。
   - 对关键组件使用 FMEA：列出失效模式、影响、检测方法。
3. **缓解策略**
   - 对每个风险制定缓解措施与 Owner。
   - 包括预防（架构设计、冗余）、检测（监控、报警）、响应（回滚、应急预案）。
4. **监控与跟踪**
   - 将风险登记到项目管理系统，设定到期复审。
   - 监控指标：RED（请求率/错误率/延迟）、USE（利用率/饱和度/错误）。
   - 定期同步状态（每周/里程碑前）。
5. **复盘与更新**
   - 发生事故时回顾风险矩阵，确认是否遗漏或低估。
   - 更新评级、缓解措施，记录改进点。

## 质量校验
- 风险登记表完整，字段无缺失。
- 缓解措施可执行，并与任务/里程碑绑定。
- 监控指标与报警策略已配置并验证。

## 失败与回滚
- 缺少 Owner：立即指定负责人，否则风险视为高优。
- 缓解措施无效：升级风险评级，重新制定方案。
- 到期未复审：列为管理缺陷，纳入 Retrospective。

## 交付物
- 风险矩阵表、FMEA 清单。
- 缓解与监控计划、执行记录。
- 复盘结论与文档更新。
