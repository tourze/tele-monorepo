# Git 取证技能

## 适用场景
- 需要追踪缺陷根源、理解模块演进、准备评审材料。
- 对比版本、生成总结、支持复盘或沟通。

> 通常在完成商业价值对齐与任务拆解后进入本技能；若当前未遇到阻塞，可直接按照以下步骤推进分析。

## 操作步骤
1. **范围定义**
   - 确定分析对象（文件、目录、提交区间）。
   - 加载相关上下文（任务、PR、事件）。
2. **历史分析**
   - `git history/ blame` 查找关键提交、作者、时间线。
   - 分析 commit message、关联 issue、分支策略。
3. **差异审查**
   - 使用 `git diff`、`--stat`、`--name-only` 获取变更概览。
   - 聚焦高风险文件（核心逻辑、安全关键）。
4. **快照与比较**
   - 结合 `method/context-snapshot` 保存会话状态，便于重现。
   - 对比多个快照或标签，识别配置/依赖差异。
5. **总结与沟通**
   - 汇总关键发现、风险、后续建议，可生成结构化报告。
   - 准备图表/表格展示提交量、作者分布、热点文件。

## 质量校验
- 分析报告包含时间线、责任人、关键提交、风险结论。
- 数据来源明确，可复现（提供命令与输出片段）。
- 对结论给出可执行建议或验证计划。

## 失败与回滚
- 数据不足：补充范围或访问权限。
- 误判根因：记录假设，等待进一步验证。
- 输出混乱：重组结构，确保信息清晰。

## 交付物
- Git 分析报告（摘要、细节、建议）。
- 快照或补丁文件。
- 后续行动项（代码修复、文档更新、分享等）。
