## 实验设计简表

| 项目 | 内容 |
| ---- | ---- |
| 实验名称 | 例如：结账页新版 CTA |
| OEC | 成功支付率（PaySuccessRate） |
| 护栏指标 | 退款率 ≤ 基线 +0.5%，错误率 ≤ 基线 ×1.2，客服投诉量无显著上升 |
| 基线与目标 | 基线 45%，目标 +3pp；单位成本从 ¥1.5 降至 ¥1.2 |
| 指标来源 | 数据仓库 `dw_fact_payment`，实时埋点 `payment_success` |
| 样本量 & 时长 | 按 80% 功效、95% 置信度、MDE=+2pp，需 N=50k 会话，预计 5 天 |
| 分流策略 | 新老用户 50/50，保持渠道、终端、版本均衡；排除高风险用户群 |
| 实验守护 | 任一护栏触发即暂停；误差预算剩余 <20% 时禁止继续放量 |
| 数据质量 | 实验开始前对照历史样本、A/A 测试 24 小时 |
| 结果判定 | 达到显著性即采纳；未达到则触发复盘并更新假设 |
